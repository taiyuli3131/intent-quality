## Longer Project Summary

`intent-quality` is a local-first collaboration quality system for people using Codex-like Agents to inspect projects, edit files, run commands, and maintain long-running work.

It turns recurring Agent collaboration failures into structured diagnosis reports, repeatable eval fixtures, public-sample safety checks, local playbook learning, and reviewable suggestions. The project focuses on mistakes that happen before ordinary output quality: misunderstood goals, unauthorized execution, stale context, unsupported premises, and wrong response modes.

The v0.3 local candidate includes diagnosis reports, collaboration-quality eval fixtures, public sync gates, profile-memory suggestion fixtures, eval review metadata, experimental adapter export drafts, playbook pages, a Codex plugin/skill entry point, and read-only validation checks. The current v0.4 P0 work adds synthetic diagnosis-calibration fixtures that check readiness, confidence, privacy blockers, and candidate gates without generating or approving real local assets.

## Share Brief

```text
intent-quality is a local-first quality layer for Codex and file-capable AI Agents. It turns collaboration failures such as misunderstood goals, unauthorized execution, stale context, unsupported premises, and wrong response modes into structured diagnosis reports, repeatable eval fixtures, reviewable suggestions, and local playbook learning. The current v0.3 candidate includes the CLI, Codex plugin/skill support, diagnosis and eval workflows, public-sample safety gates, pending profile-memory suggestions, local eval-review metadata, experimental adapter export drafts, and read-only diagnosis calibration fixtures. Its product boundary is deliberate: no hosted dashboard, no automatic upload, no automatic adoption, no default LLM-as-judge, and no mutation of rules, profiles, datasets, casebooks, rubrics, contributions, or public samples without user confirmation.
```

## Safety Emphasis

Use this when you want to emphasize trust boundaries:

```text
intent-quality is designed around a narrow safety model:

- local-first;
- public samples untrusted by default;
- profile memory as pending suggestions only;
- contribution packages are local until explicitly submitted;
- no automatic mutation of rules, profiles, datasets, casebooks, rubrics, or public sample adoption;
- check/eval/review flows are read-only.
```
